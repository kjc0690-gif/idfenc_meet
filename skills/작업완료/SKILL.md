# 작업완료 체크리스트 스킬

모든 작업/스킬 종료 시 실행하는 공통 마무리 절차.
다른 스킬에서도 작업 마무리 시 이 스킬을 호출하여 통일된 종료 절차를 따른다.

## 실행 순서

### 1단계: Git 커밋 & GitHub 푸시

1. `git status`로 변경사항 확인
2. 변경/추가된 파일이 있으면:
   - 코드, 스킬 파일, 설정 파일을 스테이징
   - `.env`, 비밀번호 등 민감 파일은 제외
   - 커밋 메시지는 한글로, 오늘 작업 내용을 요약
   - `git push origin master`로 GitHub에 푸시
3. 변경사항이 없으면 "커밋할 내용 없음"으로 건너뜀

**커밋 메시지 형식:**
```
[카테고리] 작업 내용 요약

상세 변경 내용 (있으면)

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
```

카테고리 예시: 스킬, TBM, 메일, 캘린더, 유튜브, 설정, 버그수정

### 2단계: 스킬 백업 점검 & G드라이브 백업

#### 2-1. 스킬 백업 점검 (자동)

C드라이브 스킬과 G드라이브 백업을 비교하여 누락/변경된 스킬을 자동 백업한다.

**점검 방법:**
1. C드라이브 스킬 목록 조회: `ls C:/Users/kjc06/.claude/skills/`
2. G드라이브 백업 목록 조회: `ls G:/내 드라이브/클로드작업기록/스킬백업/`
3. 비교하여:
   - **백업 없는 스킬** → 즉시 폴더 생성 후 전체 복사
   - **백업 있지만 내용 다른 스킬** → `diff`로 SKILL.md 비교, 다르면 덮어쓰기
   - **동일한 스킬** → 건너뜀
4. 스킬 하위의 scripts/, 데이터 파일도 함께 백업 (단, `__pycache__`는 제외)

**경로:**
- C드라이브 원본: `C:\Users\kjc06\.claude\skills\{스킬명}\`
- G드라이브 백업: `G:\내 드라이브\클로드작업기록\스킬백업\{스킬명}\`

**점검 출력:**
```
━━━ 스킬 백업 점검 ━━━
| 스킬명           | C드라이브 | G드라이브 | 상태       |
|-----------------|----------|----------|-----------|
| gcal-reminder   | O        | O        | 동일       |
| mail-checker    | O        | O        | 동일       |
| send-email      | O        | O        | 동일       |
| work-done       | O        | X → O    | 백업 완료   |
| youtube-shorts  | O        | O        | 업데이트    |
```

#### 2-2. 결과물 백업 확인

1. 오늘 생성/수정된 결과물(엑셀, PDF, 이미지, 영상 등) 확인
2. 코드는 GitHub에, 결과물은 G드라이브에 분리 백업 원칙
3. G드라이브 백업 경로: `G:\내 드라이브\클로드작업기록\`
4. 결과물이 이미 G드라이브에 있으면 확인만
5. C드라이브에만 있는 결과물이 있으면 복사

**백업 확인 출력:**
```
📁 G드라이브 백업 확인
- [확인] TBM 회의록 → G:\내 드라이브\클로드작업기록\TBM\현장별\...
- [확인] 숏폼 영상 → G:\내 드라이브\클로드작업기록\YouTube_Shorts\...
- (또는) 오늘 생성된 결과물 없음
```

### 3단계: 메모리 파일 정리

1. 오늘 대화에서 새로 알게 된 규칙, 설정, 사용자 피드백 확인
2. 새 메모리가 필요하면 memory 폴더에 파일 생성 + MEMORY.md 인덱스 업데이트
3. 기존 메모리 중 수정이 필요한 것이 있으면 업데이트
4. 특별히 저장할 내용이 없으면 건너뜀

**메모리 경로:** `C:\Users\kjc06\.claude\projects\C--Users-kjc06----------idfenc-meet\memory\`

### 4단계: 작업요약 브리핑

오늘 수행한 작업을 카테고리별로 정리하여 출력한다.

**출력 형식:**
```
═══════════════════════════════════════
  작업완료 브리핑 ([오늘 날짜])
═══════════════════════════════════════

━━━ Git 현황 ━━━
커밋: [커밋 메시지] (해시)

━━━ 백업 현황 ━━━

━━━ 오늘의 작업 내역 ━━━

📌 [카테고리1]
- 세부 작업 내용

```

## 다른 스킬에서 호출하는 방법

각 스킬 작업이 끝난 후, 사용자가 "작업완료" 또는 유사 표현을 하면 이 스킬이 자동 트리거된다.
개별 스킬에 종료 체크리스트를 넣지 않고, 이 스킬 하나로 통일한다.

## 주의사항

- 커밋 전 반드시 `git diff`로 변경 내용 확인
- 민감 정보(.env, 비밀번호, API 키) 커밋 금지
- 푸시 실패 시 원인 안내 후 사용자에게 대안 제시
- 메모리는 미래 대화에 유용한 것만 저장 (일시적 정보는 저장 안 함)
- 브리핑은 간결하게, 한눈에 파악 가능하도록
