# 대화 인사이트 기록 스킬

사용자가 "기록해 줘"라고 할 때, 그 지점부터 **역으로** 대화 내용을 분석하여 중요한 인사이트를 마크다운 문서로 자동 저장합니다.

## 당신의 역할

다음 단계를 따라 주세요:

### 1단계: 대화 분석
사용자가 "기록해 줘"라고 한 지점에서 **그 이전 대화들을 역순으로** 검토합니다.
- 처음부터가 아니라, 현재 시점에서 **뒤로 가면서** 관련 내용을 모읍니다.
- 보통 최근 5~10개 메시지가 핵심이지만, 필요하면 더 거슬러 올라갑니다.

### 2단계: 4가지 카테고리로 분류

대화에서 다음을 찾아 정리합니다:

**1) 주요 결정사항 / 합의사항**
- "~하기로 했다"
- "~를 선택했다"
- "다음부터는 ~로 진행하자"

**2) 학습한 것들**
- 새로 알게 된 사실
- 교훈
- 인사이트
- "아, 그렇구나!"라는 발견

**3) 문제점과 해결책**
- 발견된 문제나 이슈
- 제시된 해결책
- 개선안

**4) 행동 아이템 (To Do)**
- 앞으로 해야 할 일
- 책임자 (있으면)
- 마감일 (있으면)

### 3단계: 마크다운 문서 생성

다음 템플릿을 사용하여 마크다운을 작성합니다. 비어있는 섹션은 생략 가능합니다:

```markdown
# 대화 기록

**기록 날짜**: 2026-03-12
**기록 시간**: 14:35
**주제**: [자동으로 대화 주제 파악하여 입력]

---

## 📌 주요 결정사항 / 합의사항

- [결정사항 1]
- [결정사항 2]

---

## 💡 학습한 것들

- [학습점 1]
- [학습점 2]

---

## ⚠️ 문제점과 해결책

### 문제: [문제 설명]
**해결책**: [제시된 해결책]

---

## ✅ 행동 아이템 (To Do)

- [ ] [할 일 1] - [책임자, 있으면]
- [ ] [할 일 2] - [기한, 있으면]

---

## 📝 추가 메모

[필요시 추가 정보나 맥락]
```

### 4단계: 파일 저장

- 파일명 형식: `insights_YYYY-MM-DD_HHMMSS.md`
  - 예: `insights_2026-03-12_143500.md`
- 저장 경로: 사용자의 프로젝트 디렉토리 또는 지정된 경로
  - 경로가 명확하지 않으면 현재 작업 디렉토리에 저장

## 좋은 예시

**사용자 말**: "지금까지 논의한 거 기록해 줘"

당신의 접근:
1. 현재부터 역으로 대화 훑기
2. "API 설계는 REST로 가기로 결정", "데이터베이스는 PostgreSQL 선택", "테스트는 pytest 사용" → 주요 결정사항
3. "NoSQL의 스케일링이 문제다" → 학습점
4. "동시성 처리가 병목" → 문제점, "캐싱 도입" → 해결책
5. "테스트 코드 작성", "배포 스크립트 준비" → 할 일
6. 마크다운 작성 후 파일 저장

## 중요 포인트

- **역방향 분석**: "기록해 줘"가 나온 시점에서 **이전 대화**를 분석합니다.
- **현재 시간 자동 포함**: 항상 기록 시간을 최상단에 표시합니다.
- **명확성**: 모호한 내용은 피하고, 대화에서 직접 나온 표현을 최대한 사용합니다.
- **비어있는 섹션 생략**: 카테고리에 내용이 없으면 그 섹션을 빼도 됩니다.
