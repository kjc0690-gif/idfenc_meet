# TBM 회의록 자동 작성 스킬

건설현장에서 매일 작성하는 안전 TBM(Tool Box Meeting) 회의록을 자동으로 만들어주는 스킬입니다.

## 작동 방식

1. **템플릿 복사 방식**: `assets/TBM_회의록_템플릿.xlsx` 파일을 복사한 뒤, 데이터만 채워넣습니다. 절대 엑셀 양식을 새로 만들지 마세요.
2. **자동 생성 항목**: 위험요인, 대책, 중점위험요인, 중점대책은 작업명/작업내용의 키워드를 분석해서 자동으로 채웁니다.
3. **사진 자동 선택**: 사진 폴더를 지정하면 오늘 날짜 사진만 골라서 사진대지에 넣습니다.

## 사용자에게 물어볼 것 (순서대로)

1. **참석자** — 쉼표로 구분 (예: 최준호, 한웅철, 남만우). 첫 번째 사람이 TBM 리더가 됩니다.
2. **현장명** — 예: 온양농협 대규모점포(하나로마트) 신축공사
3. **소속(회사명)** — 예: (주)아이디에프이앤씨
4. **작업명** — 예: 습식방수공사
5. **TBM 장소** — 예: 지상1층
6. **작업내용** — 사용자가 직접 작성

진행자 입력은 없습니다. 참석자 첫 번째가 자동으로 리더입니다.

## 위험요인/대책 자동 생성 (키워드 매칭)

작업명과 작업내용을 합쳐서 아래 키워드를 찾고, 해당하는 위험요인 3개와 대책 3개를 자동 생성합니다.

| 키워드 | 위험요인 ①②③ | 대책 ①②③ |
|--------|-------------|----------|
| 높이 | 추락위험 / 낙하위험 / 전도사고 | 안전대착용·안전로프 / 슬링벨트확인 / 바닥정리·미끄럼방지 |
| 전기 | 감전위험 / 화재위험 / 전도사고 | 절연도구·누전차단기 / 화기금지·소화기 / 바닥정리 |
| 폐쇄 | 산소부족 / 질식위험 / 전도사고 | 환기·산소측정 / 산소호흡기·감시원 / 바닥정리 |
| 중장비 | 협착사고 / 낙하위험 / 전도사고 | 신호원배치 / 슬링벨트확인 / 바닥정리 |
| 방수 | 추락위험 / 낙하위험 / 전도사고 | 안전대착용 / 슬링벨트확인 / 미끄럼방지 |
| 도장 | 중독위험 / 화재폭발 / 전도사고 | 마스크·환기 / 화기금지·정전기제거 / 바닥정리 |
| 견출 | 추락위험 / 낙하위험 / 전도사고 | 안전대착용 / 슬링벨트확인 / 미끄럼방지 |
| 철거 | 추락위험 / 낙하위험 / 전도사고 | 안전대착용 / 슬링벨트확인 / 미끄럼방지 |
| 용접 | 화재폭발 / 화상위험 / 전도사고 | 소화기·가연물제거 / 방열복·차광막 / 바닥정리 |
| 굴착 | 토사붕괴 / 매설물파손 / 전도사고 | 흙막이·지반조사 / 매설물탐사 / 바닥정리 |
| 타일 | 추락위험 / 낙하위험 / 전도사고 | 안전대착용 / 슬링벨트확인 / 미끄럼방지 |
| 배관 | 협착사고 / 낙하위험 / 전도사고 | 신호원배치 / 슬링벨트확인 / 바닥정리 |

매칭되는 키워드가 없으면 기본값(추락/낙하/전도)을 사용합니다.

## 엑셀 셀 위치 (TBM 회의록 시트 — 첫 번째 시트)

| 셀 | 내용 | 형식 |
|----|------|------|
| B3 | TBM 일시 | `2026 년 03 월 18 일, 09:00~ 작업날짜와 동일함 (예→■, 아니오□)` |
| B4 | 현장명+작업명 | `현장명 : OOO    작업명 : OOO` |
| B5 | 작업내용 | 사용자 입력 그대로 |
| B6 | TBM장소 | 사용자 입력 |
| A8~A10 | 위험요인 ①②③ | `① 상부 작업자 하부 추락 위험` |
| D8~D10 | 대책 ①②③ | `① 비계 설치등 상부 작업자 안전대 착용...` |
| C11 | 중점위험요인 선정 | 예: `추락방지` |
| C12 | 중점위험요인 대책 | 예: `안전대 착용 및 안전로프 사전 설치` |
| A13 | TBM 리더확인 | `TBM 리더확인  · 소속 OOO  · 직책 반장  · 성명 OOO (서명)` |
| A16~A18 | 안전조치 확인 | 위험요인 ①②③ 반복 |
| A24~ | 참석자 이름 | 3열(A,C,E) 배치, 세로 방향 |

## 엑셀 셀 위치 (사진대지 시트 — 두 번째 시트)

| 셀 | 내용 |
|----|------|
| A2 | `현장명 : OOO` |
| A3 | `작업명 : 회사명 (작업명)` |
| A4 | `날   짜 : 2026. 03. 18 (수)` |
| B6 | 공종 (= 작업명) |
| B7 | 위치 (= TBM장소) |
| A5 | 큰 사진 (첫 번째, 500x380 맞춤) |
| A8 | 왼쪽 아래 사진 (두 번째, 240x250 맞춤) |
| C8 | 오른쪽 아래 사진 (세 번째, 240x250 맞춤) |

## 사진 처리 규칙

사진 폴더가 지정되면 오늘 날짜 사진만 자동 선택합니다:

1. 파일명에 오늘 날짜 포함 여부 (20260318, 2026-03-18, 2026.03.18, 0318)
2. 파일 수정일이 오늘인지
3. EXIF 촬영일이 오늘인지

사진은 원본을 유지하고, 엑셀에 넣을 때만 비율 유지하면서 크기를 조정합니다.

## 실행 방법

`scripts/tbm_recorder.py`를 Python으로 실행하면 GUI 프로그램이 뜹니다. 필요한 패키지: `openpyxl`, `Pillow`, `tkinter`

```bash
pip install openpyxl Pillow
python scripts/tbm_recorder.py
```

## 중요 규칙

- 엑셀 템플릿은 절대 새로 만들지 말 것. 반드시 `assets/TBM_회의록_템플릿.xlsx`를 복사해서 사용
- 사용자에게 설명할 때는 초등학생도 알 수 있도록 쉽게
- 전문 용어(터미널, 환경변수 등) 사용 금지
- 회의록은 `회의록/` 폴더에 `TBM_회의록_날짜.xlsx` 형식으로 저장
