# 안전 TBM 회의록 자동 작성 스킬

## 최우선: 프로젝트(현장) 등록부 확인

**모든 TBM 작업 전에 반드시 이 파일을 먼저 읽는다:**
```
G:\내 드라이브\클로드작업기록\TBM\projects.json
```

이 파일에 현장별 정보(현장명, 참석자, 작업명, 폴더 경로 등)가 등록되어 있다.

### 작업 흐름
1. `projects.json` 읽기
2. 사용자가 말한 현장명과 매칭
3. 매칭되면 → 해당 현장 정보로 자동 진행
4. 매칭 안 되면 → 새 현장 등록 (사용자에게 정보 확인 후 `projects.json`에 추가)

## 프로그램 위치 (Google Drive - 모든 PC 공유)

```
G:\내 드라이브\클로드작업기록\TBM\
```

### 폴더 구조
```
TBM/
├── projects.json              ← 현장 등록부 (항상 먼저 읽을 것!)
├── tbm_recorder.py            ← GUI 프로그램 (단건)
├── batch_create.py            ← 일괄 생성 (다건)
├── 입력이력.json              ← 자동완성용
├── 20260318/TBM_회의록_20260316.xlsx  ← 엑셀 템플릿
├── 현장별/
│   ├── 농협은행울산영업부/
│   │   ├── 회의록/   ← TBM_회의록_20260316.xlsx ~ 20260321.xlsx
│   │   └── 사진/     ← 20260316.jpg ~ 20260321.jpg
│   └── (새 현장 폴더 자동 생성)
├── 회의록/            ← (구) 하위호환
└── 사진/              ← (구) 하위호환
```

## 실행 방법

### 패키지 확인
```bash
python -c "import openpyxl, PIL" 2>/dev/null || python -m pip install openpyxl Pillow
```

### GUI 단건 작성
```bash
cd "G:/내 드라이브/클로드작업기록/TBM" && python tbm_recorder.py
```

### 스크립트 일괄 작성
`batch_create.py`의 상수를 현장 정보로 수정 후:
```bash
cd "G:/내 드라이브/클로드작업기록/TBM" && python batch_create.py
```

## 고정 정보

- **회사명**: `(주)아이디에프이앤씨` (코드에 고정)
- **TBM 시간**: GUI=현재시간, 일괄=`08:00`
- **TBM 리더**: 참석자 첫 번째
- **직책**: 반장

## 새 현장 등록 절차

사용자가 새 현장 TBM을 요청하면:
1. 사용자에게 확인: 현장명, 참석자, 작업명, TBM장소, 작업내용
2. `projects.json`에 새 프로젝트 추가
3. `현장별/{현장이름}/회의록/`, `현장별/{현장이름}/사진/` 폴더 생성
4. TBM 회의록 생성

| 방수 | 추락, 낙하, 전도 | 전도방지 |
| 도장 | 중독, 화재/폭발, 전도 | 중독방지 |
| 견출 | 추락, 낙하, 전도 | 전도방지 |

## 엑셀 셀 매핑 (1페이지: TBM 회의록)

| 셀 | 내용 |
|----|------|
| B3 | `YYYY 년 MM 월 DD 일, HH:MM~ 작업날짜와 동일함 (예→■, 아니오□)` |
| B4 | `현장명 : {현장명}    작업명 : {작업명}` |
| B5 | 작업내용 |
| B6 | TBM장소 |
| A8~A10 | ①②③ 잠재위험요인 |
| D8~D10 | ①②③ 대책 |
| C11 | 중점위험요인 |
| C12 | 중점대책 |
| A13 | `TBM 리더확인 · 소속 {회사} · 직책 반장 · 성명 {리더} (서명)` |
| A16~A18 | ①②③ 안전조치 확인 |
| A24~E29 | 참석자 (3열 세로 배치) |

## 엑셀 셀 매핑 (2페이지: 사진대지)

| 셀 | 내용 |
|----|------|
| A2 | `현장명 : {현장명}` |
| A3 | `작업명 : {회사} ({작업명})` |
| A4 | `날   짜 : YYYY. MM. DD (요일)` |
| B6 | 공종 |
| B7 | 위치 |
| A5 | 사진1 (500x380) |
| A8 | 사진2 (240x250) |
| C8 | 사진3 (240x250) |

## 사진 규칙

- 파일명: `YYYYMMDD.jpg`
- 저장: 해당 현장의 `사진/` 폴더
- GUI: 오늘 날짜만 자동 선택, 없으면 사진 없이 진행
- 일괄: 날짜별 파일명 매칭
