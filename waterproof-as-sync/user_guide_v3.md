## 1. 시작하기

### 1.1 필요한 환경
- **브라우저**: Chrome, Edge, Safari, Firefox (최신 버전)
- **용량**: LocalStorage 5MB 이상 여유공간
- **해상도**: 1024x768 이상 권장
- **인터넷**: 불필요 (완전 오프라인 동작)

### 1.2 설치 방법
```bash
# 방법 1: 다운로드 링크에서 파일 다운로드
1. 다운로드 링크 클릭
2. waterproofing_as_v3_optionA.html 저장
3. 더블클릭하여 브라우저에서 실행

# 방법 2: AI Drive에서 직접 실행
1. AI Drive > /방수명장/AS대응프로세스/ 폴더 이동
2. waterproofing_as_v3_optionA.html 클릭
3. "다운로드" 또는 "미리보기" 선택
```

### 1.3 첫 실행
- 자동으로 **AS 번호**가 생성됩니다 (예: AS-20260401-0001)
- 모든 입력 데이터는 **실시간 자동 저장**됩니다
- 브라우저를 닫아도 **데이터가 유지**됩니다

## 3. 단계별 사용법

### Step 1: 📋 공문 접수

#### 3.1.1 공문 사진 업로드
```
방법 1: 드래그앤드롭
- 파일 탐색기에서 사진을 드래그하여 업로드 영역에 드롭

방법 2: 클릭 업로드
- "클릭하거나 드래그하여 공문 사진 업로드" 영역 클릭
- 파일 선택 창에서 사진 선택 (JPG, PNG, PDF)

방법 3: 모바일
- 업로드 영역 터치
- 카메라 또는 갤러리 선택
```

#### 3.1.2 AI 검토 의견 확인
1. 사진 업로드 후 **AI 분석 실행** 버튼 클릭
2. AI가 자동으로 분석한 의견이 표시됩니다
   - 공문 내용 요약
   - 주요 포인트
   - 권장 조치 사항

> **참고**: 현재 버전은 Mock AI로, 샘플 텍스트가 표시됩니다.

#### 3.1.3 현장 정보 입력
```yaml
현장명: [필수]
  예시: "OO아파트 101동 옥상 방수공사"
  
현장 개요: [필수]
  작성 내용:
  - 건물 준공연도
  - 기존 방수 이력
  - 균열 발생 원인
  - 기타 특이사항
  
  예시:
  "2010년 준공, 기존 우레탄 도막방수 시공.
   파라펫 상단 균열로 인한 우천 시 누수 발생.
   최근 2년간 부분보수 이력 있으나 재발."
```

#### 3.1.4 다음 단계 이동
- **다음: 기본정보 입력 →** 버튼 클릭
- 현장명과 현장개요가 모두 입력되어야 이동 가능

### Step 2: 📝 기본정보 입력

#### 3.2.1 구조물 정보
| 항목 | 필수 | 예시 |
|------|:----:|------|
| 건물 유형 | ✅ | 아파트, 단독주택, 상가건물 등 |
| 시공 부위 | ✅ | 옥상, 외벽, 지하실 등 |
| 시공 면적 | ⬜ | 150 (㎡) |
| 준공 연도 | ⬜ | 2010 |

#### 3.2.2 누수 증상
```yaml
주요 증상: [필수]
  작성 포인트:
  - 누수 위치 (구체적으로)
  - 발생 시점 (언제부터)
  - 증상 상세 (어떻게)
  
  예시:
  "101동 10층 거실 천장에서 우천 시마다 물방울 떨어짐.
   3월 15일 첫 발생 후 비올 때마다 재발.
   천장 중앙부에 물얼룩 직경 약 30cm."

기존 방수 공법: [선택]
  예시: "우레탄 도막방수", "시트방수" 등

최초 발생일: [선택]
  달력에서 선택
```

#### 3.2.3 고객 정보
| 항목 | 필수 | 예시 |
|------|:----:|------|
| 고객명 | ✅ | 홍길동 |
| 연락처 | ✅ | 010-1234-5678 |
| 주소 | ⬜ | 서울시 강남구... |

### Step 3: ⚡ 긴급도 판단

#### 3.3.1 체크리스트 작성
5개 항목을 체크하면 **자동으로 점수 계산** 및 긴급도가 표시됩니다.

| 체크 항목 | 점수 |
|-----------|:----:|
| ☑ 누수가 실내로 들어와 생활에 지장이 있음 | +3점 |
| ☑ 구조적 균열이 발견됨 | +2점 |
| ☑ 누수 발생 후 24시간 이내 | +2점 |
| ☑ 외벽 또는 옥상에서 육안 균열 확인 | +1점 |
| ☑ 우천 시에만 발생 | +1점 |

#### 3.3.2 긴급도 결과
```yaml
총점 5점 이상:
  긴급도: 높음 (긴급)
  대응: 48시간 내 현장 방문

총점 3-4점:
  긴급도: 보통
  대응: 1주일 내 현장 방문

총점 0-2점:
  긴급도: 낮음
  대응: 예약 방문
```

### Step 4: 📅 활동 관리

#### 3.4.1 새 활동 추가
1. **+ 새 활동 추가** 버튼 클릭
2. 활동 유형 선택:
   - 🔍 현장 방문
   - 💬 회의
   - ⚒️ 시공

#### 3.4.2 유형별 입력 양식

##### A. 현장 방문
```yaml
날짜: [필수] 2026-04-01
시간: [선택] 14:00
활동 내용:
  "옥상 파라펫 균열 확인.
   드레인 주변 들뜸 현상 발견.
   임시 방수 테이프 조치."
사진 첨부: 현장 사진 업로드
```

##### B. 회의
```yaml
날짜/시간: [필수]
활동 내용: 회의 안건

회의록:
  참석자: [Enter로 추가]
    - 홍길동 (건물주)
    - 김기술 (방수명장)
    - 이시공 (시공사)
  
  주요 결정사항:
    "우레탄 도막방수 전면 재시공 결정.
     예산 3,000만원 승인."
  
  보류 사항:
    "최종 착공일은 기상 확인 후 결정"
  
  액션 아이템:
    할일: "견적서 제출"
    담당자: "김기술"
    기한: 2026-04-05
    
    [+ 액션 아이템 추가] 버튼으로 여러 개 추가 가능

사진 첨부: 회의 현장 사진 (선택)
```

##### C. 시공
```yaml
날짜/시간: [필수]
활동 내용: 시공 개요

시공일지:
  기상: ☀️ 맑음 (드롭다운 선택)
  투입 인원: 5명
  
  작업 내용:
    "바탕면 고압세척 및 건조.
     프라이머 1차 도포 완료.
     균열부 V-커팅 및 실링재 충진."
  
  사용 자재:
    "프라이머 10L (㈜OO산업)
     우레탄 방수재 30kg (㈜△△케미칼)
     실링재 5개"
  
  진척도: 20% (슬라이더 조절)
    └─ 자동으로 프로그레스바 표시
  
  특이사항:
    "바탕면 함수율 양호(5%).
     일부 구간 재고압세척 필요."

사진 첨부: 작업 사진 (Before/After 권장)
```

#### 3.4.3 활동 저장
- **저장** 버튼 클릭
- 활동 목록에 카드 형태로 추가됨
- 무제한 활동 추가 가능

## 4. 고급 기능

### 4.1 사진 관리

#### 4.1.1 사진 업로드 팁
```yaml
권장 사항:
- 파일 크기: 5MB 이하
- 해상도: 1920x1080 권장
- 포맷: JPG, PNG
- 개수: 제한 없음 (단, LocalStorage 5MB 제한)

주의 사항:
- 사진 많을수록 백업 파일 크기 증가
- 10장 이상 시 리사이징 권장
```

#### 4.1.2 사진 삭제
```
1. 사진 미리보기 우측 상단 [×] 버튼 클릭
2. 삭제 후 자동 저장
```

### 4.2 회의록 작성 꿀팁

#### 4.2.1 참석자 빠르게 추가
```
1. 참석자 입력란에 이름 입력
2. Enter 키 누르기 (자동 태그 생성)
3. 반복
```

#### 4.2.2 액션 아이템 관리
```yaml
필수 정보:
- 할일: 구체적으로 작성
- 담당자: 실명 기재
- 기한: 달력에서 선택

예시:
할일: "견적서 3사 비교표 작성 후 이메일 발송"
담당자: "김기술"
기한: 2026-04-05
```

### 4.3 시공일지 작성 꿀팁

#### 4.3.1 진척도 표시
```
슬라이더로 0~100% 조절
└─ 자동으로 프로그레스바 업데이트
└─ 색상으로 진행 상황 시각화
```

#### 4.3.2 자재 입력 형식
```yaml
권장 형식:
"품목명 수량 (제조사/모델)"

예시:
"우레탄 방수재 30kg (㈜OO산업 WP-2000)
 프라이머 10L (㈜△△케미칼 PR-100)
 실링재 5개 (□□실업 SL-300)"
```

## 6. 문제해결

### 6.1 자주 발생하는 문제

#### Q1. 데이터가 사라졌어요
```yaml
원인:
- 브라우저 캐시 삭제
- 다른 브라우저로 실행
- 시크릿 모드 사용

해결:
1. 백업 JSON 파일 확인
2. Google Drive에 업로드한 파일 확인
3. 동일 브라우저 재실행

예방:
- 일일 백업 다운로드
- Google Drive 수동 업로드
```

#### Q2. 사진이 너무 많아서 느려요
```yaml
원인:
- 사진 파일 크기 과다
- Base64 인코딩 용량 증가

해결:
1. 사진 리사이징 (1920x1080 이하)
2. JPG 품질 낮춤 (80%)
3. 불필요한 사진 삭제

권장:
- 현장 방문당 10장 이내
- 시공 일차당 20장 이내
```

#### Q3. 다음 단계로 이동이 안 돼요

```yaml
  ✅ 주요 증상

  ✅ 연락처

해결:
- 빨간색 별표(*) 항목 모두 입력
```

### Q1. 여러 AS를 동시에 관리할 수 있나요?
```yaml
현재 버전: 1개 케이스만 관리
└─ LocalStorage에 1개 케이스만 저장

차후 업데이트:
- 여러 AS 케이스 동시 관리
- 케이스 목록 UI 추가
- 케이스 전환 기능
```

### Q2. 모바일에서도 사용 가능한가요?
```yaml
✅ 사용 가능 (반응형 디자인)

권장 환경:
- iOS Safari (14+)
- Android Chrome (90+)

주의:
- 사진 많을 시 느려질 수 있음
- 큰 화면(태블릿) 권장
```

### Q3. 오프라인에서 사용 가능한가요?
```yaml
✅ 완전 오프라인 동작

필요 없는 것:
- 인터넷 연결
- 서버 설치
- 데이터베이스

필요한 것:
- 브라우저만 있으면 OK
```

### Q4. PDF로 내보낼 수 있나요?
```yaml
현재 버전: JSON만 지원

차후 업데이트:
- PDF 자동 생성
- Excel 내보내기
- 이메일 자동 발송
```

### Q5. 팀원들과 공유하려면?
```yaml
방법 1: JSON 파일 공유
  1. JSON 백업 다운로드
  2. 이메일/메신저로 전송
  3. 팀원이 동일 HTML 파일 실행 후 수동 복원 (차후 지원)

방법 2: Google Drive 공유
  1. JSON 백업을 Google Drive 업로드
  2. 폴더 공유 설정
  3. 팀원이 다운로드 후 확인

방법 3: 화면 공유
  - 브라우저 화면을 Zoom/Teams로 공유
```

### Q6. 데이터 보안은 어떻게 되나요?
```yaml
저장 위치:
- 사용자 로컬 브라우저 (LocalStorage)
- 외부 서버로 전송되지 않음

주의 사항:
- 공용 PC 사용 시 로그아웃 후 브라우저 데이터 삭제
- 민감 정보는 백업 파일 암호화 권장
```

### Q7. 데이터 용량 제한은?
```yaml
LocalStorage 제한:
- 브라우저당 약 5~10MB

사진 개수 권장:
- 총 사진: 50장 이내
- 현장 방문당: 10장 이내
- 시공 일차당: 20장 이내

초과 시:
- 일부 데이터 저장 실패 가능
- 백업 JSON 다운로드 후 삭제 권장
```
