# (주)아이디에프이앤씨 AS대응 프로세스

## 역할

방수/바닥 공사 하자 AS를 **대화형**으로 접수·처리·관리하는 스킬.
사용자가 정보를 말하면 클로드가 처리한다. 사용자가 직접 입력하는 것은 최소화.

## 현장별 폴더 구조

```
현장관리\AS대응프로세스\현장별\{현장명}\
├── 공문\              ← 접수 공문 PDF 보관
├── 사진\
│   ├── {접수일}\      ← 접수일 기준 폴더 (예: 20260403)
│   │   ├── 전\        ← 작업 전 / 하자 사진
│   │   └── 후\        ← 작업 중 + 작업 후 사진
│   ├── {재접수일}\    ← 재접수 시 새 폴더 추가
│   │   ├── 전\
│   │   └── 후\
│   └── ...
└── 작업일지\          ← 접수대장, 작업일지, 완료보고서
```

**사진 폴더 규칙:**
- 접수 건마다 접수일(YYYYMMDD) 폴더 생성
- 같은 현장 재접수 시 새 날짜 폴더 추가
- 전/ = 작업 전 + 하자 사진, 후/ = 작업 중 + 작업 후 사진

## 대화형 운영 방식

### 원칙
- **클로드가 하는 것**: 하자 사진 분석, 검토의견 작성, 데이터 입력, 엑셀 생성(사진 삽입 포함), 메일 발송, 캘린더 등록, 백업
- **사용자가 하는 것**: 사진을 G드라이브 폴더에 넣기, 정보를 말로 전달, 최종 승인

### 하자 사진 분석 (클로드 코드에서 직접)
- 사용자가 G드라이브 사진 폴더에 사진을 넣으면 클로드가 직접 분석
- "하자 분석해줘" → 사진 읽기 + 원인 분석 + 보수방법 추천 + 서류 작성까지 한번에
- **채팅에 따로 사진 올릴 필요 없음** (사진 작업 1번으로 끝)

분석 결과 형식:
```
[하자 분석]
1. 하자 유형: (균열/들뜸/박리/누수/변색 등)
2. 추정 원인: (시공불량/하중과다/수축팽창/습기/설계문제 등)
3. 심각도: (경미/보통/심각)
4. 보수 방법: 1안, 2안
5. 주의사항
```
→ 분석 결과를 접수대장 '하자검토의견'란에 자동 입력

### 채팅 프로젝트 연동 (전문 상담용)
- claude.ai 채팅 "(주)아이디에프이앤씨 현장관리" 프로젝트는 **전문 상담용**으로 사용
- 견적 검토, 계약 검토, 법적 판단, 클레임 대응 등 심층 분석이 필요할 때만 활용
- 채팅에서 받은 의견을 복붙하면 클로드가 서류에 자동 입력
- 예: "하자의견은 OOO이야" → 접수대장/완료보고서 검토의견란에 입력

### 역할 분담 정리

| 작업 | 클로드 코드 (여기) | 채팅 (claude.ai) |
|------|-------------------|-----------------|
| 하자 사진 분석 | **여기서 처리** | 사용 안 함 |
| 서류 작성 | **여기서 처리** | 사용 안 함 |
| 메일/캘린더 | **여기서 처리** | 사용 안 함 |
| 견적 검토 | 가능 | 심층 분석 시 |
| 계약/법적 판단 | 가능 | 심층 분석 시 |
| 클레임 대응 | 가능 | 전략 상담 시 |

## AS 처리 업무 흐름

```
① AS접수 → ② 현장방문 → ③ 시공(작업일지) → ④ 완료보고
```

### 1단계: AS 접수 - "AS 접수해줘"

**입력 방법:**
- 공문 PDF 첨부: PDF에서 자동 추출 → 부족한 항목만 추가 질문
- 말로 전달: 클로드가 아래 항목을 하나씩 질문

**중요: 접수 시 클로드는 반드시 아래 항목을 확인하고, 빠진 항목이 있으면 질문해서 채운다.**
**모든 항목이 채워져야 접수대장 엑셀에 입력할 수 있다.**

**접수대장 필수 항목 (15개):**

| 순번 | 항목 | 질문 예시 | 비고 |
|------|------|----------|------|
| 1 | 현장명 | "어느 현장인가요?" | 필수 |
| 2 | 접수일자 | 오늘 날짜 자동 | 자동 |
| 3 | 접수경로 | "어떻게 접수됐나요? (공문/전화/메일/현장발견)" | 필수 |
| 4 | 하자부위 | "하자 위치가 어디인가요? (예: 1층 하역장 바닥)" | 필수 |
| 5 | 하자내용 | "어떤 하자인가요? (예: 바닥 균열, 에폭시 들뜸)" | 필수 |
| 6 | 하자검토의견 | "검토 의견 있으세요? 없으면 공란으로 갈게요" | 공란 가능 |
| 7 | 긴급도 | "긴급인가요, 일반인가요?" | 필수 |
| 8 | 예상공사비 | "견적이나 예상 금액 있으세요?" | 공란 가능 |
| 9 | 발주처 | "발주처(회사명)가 어디인가요?" | 필수 |
| 10 | 담당자 | "담당자 이름이요?" | 필수 |
| 11 | 연락처 | "담당자 연락처(전화번호/이메일)?" | 필수 |
| 12 | 처리상태 | "접수" 자동 입력 | 자동 |
| 13 | 처리완료일 | 공란 (완료 시 입력) | 자동 |
| 14 | 예정일 | "보수일 또는 점검일이 언제인가요?" | 공란 가능 → 있으면 캘린더 등록 |
| 15 | 비고 | "특이사항 있으세요?" | 공란 가능 |
| 16 | 하자사진 | "사진 폴더에 넣었나요?" | 공란 가능 |

**질문 규칙:**
- 한번에 여러 정보를 말하면 추가 질문 없이 바로 처리
- 공문 PDF에서 추출한 정보는 다시 묻지 않음
- "공란 가능" 항목은 없으면 넘어감
- 모든 필수 항목이 채워지면 접수 확인서 표시 → 승인 요청
- **승인 후** cases.json 저장 + 접수대장 엑셀 작성 + 사진 삽입

**클로드가 자동으로:**
1. 현장별 폴더 생성 (공문/, 사진/{접수일}/전/, 사진/{접수일}/후/, 작업일지/)
2. cases.json에 케이스 등록
3. 공문 PDF → 현장별/{현장명}/공문/ 에 이름 변경 후 저장
4. AS접수대장 엑셀 업데이트
5. 미팅/방문 일정 → 구글 캘린더 등록
6. 접수 확인서 표시 → 승인 요청

### 2단계: 현장 방문 - "현장 방문 등록해줘"

클로드가 물어볼 것:
1. 방문 일시
2. 방문자
3. 준비물 (하자 내용 기반 자동 추천)

클로드가 자동으로:
- 구글 캘린더에 일정 등록
- 담당자에게 방문 안내 메일 발송 (승인 후)

### 3단계: 시공 - "작업일지 써줘"

**사전 준비:** 사진을 현장별/{현장명}/사진/{접수일}/전/, 후/ 폴더에 넣기

**작업일지 필수 항목 (13개):**

| 순번 | 항목 | 질문 예시 | 비고 |
|------|------|----------|------|
| 1 | 작업일자 | "작업 날짜가 언제인가요?" | 필수 (오늘 자동 가능) |
| 2 | 현장명 | cases.json에서 자동 | 자동 |
| 3 | 케이스ID | cases.json에서 자동 | 자동 |
| 4 | 작업부위 | "어디 작업했나요?" | 필수 |
| 5 | 작업내용 | "오늘 뭐 했나요?" | 필수 |
| 6 | 작업자 | "누가 작업했나요? (이름, 직종)" | 필수 (최소 1명, 최대 5명) |
| 7 | 금일 작업률 | "오늘 몇 % 했나요?" | 공란 가능 |
| 8 | 누적 작업률 | "전체 몇 % 됐나요?" | 공란 가능 |
| 9 | 내일 계획 | "내일은 뭐 할 예정이에요?" | 공란 가능 |
| 10 | 특이사항 | "특이사항 있나요?" | 공란 가능 |
| 11 | 작성자 | "작성자 이름?" | 필수 |
| 12 | 발주처 담당자 | cases.json에서 자동 | 자동 |
| 13 | 사진 | "사진 폴더에 넣었나요?" | 공란 가능 |

**클로드가 자동으로:**
- AS작업일지 엑셀 생성 (3개 시트: 작업일지 + 작업 전 사진 + 작업 후 사진)
- 사진 폴더에서 사진 읽어서 엑셀에 자동 삽입
- 현장별/{현장명}/작업일지/ 에 저장

**같은 부위 여러번 작업 가능** - 작업일지는 날짜별로 생성 (AS작업일지_20260406.xlsx)

### 4단계: 완료 - "완료보고 써줘"

**클로드가 자동으로:**
- cases.json에서 접수~시공 이력 전부 불러오기
- AS완료보고서 엑셀 생성 (보수이력 1차~5차 자동 기재)
- 작업 전/후 사진 자동 삽입
- 완료 안내 메일 발송 (승인 후)
- cases.json 상태 "완료"로 변경

## AS 현황 조회

사용자: "AS 현황 보여줘"
→ cases.json 읽어서 표로 표시

## 엑셀 양식 구성

### AS접수대장 (목록형)
- 시트1: 접수대장 (접수번호, 현장명, 접수일자, 접수경로, 하자부위, 하자내용, 하자검토의견, 긴급도, 예상공사비, 발주처, 담당자, 연락처, 처리상태, 처리완료일, 비고)
- 시트2: 접수사진 (사진 6장)

### AS작업일지 (건별/날짜별)
- 시트1: 작업일지 (기본정보, 작업내용, 작업자 5명, 진행상황, 특이사항, 확인)
- 시트2: 작업 전 사진 (사진 6장)
- 시트3: 작업 후 사진 (사진 6장)

### AS완료보고서 (건별)
- 시트1: 완료보고서 (기본정보, 하자개요, 보수작업이력 1~5차, 최종보수결과, 특이사항, 확인)
- 시트2: 작업 전 사진 (사진 6장)
- 시트3: 작업 후 사진 (사진 6장)
