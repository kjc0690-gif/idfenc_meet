# TBM 자동화 프로젝트

## 핵심: 프로젝트(현장) 기반 운영

여러 현장을 운영하므로 **반드시 `projects.json`을 먼저 읽고** 현장 정보를 확인한 후 작업한다.

```
G:\내 드라이브\클로드작업기록\TBM\projects.json
```

### 사용자가 "OO현장 TBM 작성해줘" 하면:
1. `projects.json`에서 현장명 검색
2. 해당 현장의 참석자, 작업명, 폴더 경로 등 자동 로드
3. 해당 현장 폴더(`현장별/현장이름/회의록/`)에 저장
4. 새 현장이면 `projects.json`에 등록 + 폴더 생성

### 사용자가 "회의록 보여줘" 하면:
1. `projects.json`에서 현장 목록 확인
2. 해당 현장의 `현장별/현장이름/회의록/` 폴더 조회

## 폴더 구조

```
G:\내 드라이브\클로드작업기록\TBM\
├── projects.json              ← 현장 등록부 (항상 먼저 읽을 것)
├── tbm_recorder.py            ← GUI 프로그램
├── batch_create.py            ← 일괄 생성 스크립트
├── 입력이력.json              ← 자동완성용
├── 20260318\TBM_회의록_20260316.xlsx  ← 엑셀 템플릿 (절대 수정 금지)
├── 현장별\
│   ├── 농협은행울산영업부\
│   │   ├── 회의록\            ← 이 현장의 TBM 회의록
│   │   └── 사진\              ← 이 현장의 사진
│   └── (새 현장 추가 시 폴더 자동 생성)
├── 회의록\                    ← (구) 저장 폴더 (하위호환)
└── 사진\                      ← (구) 사진 폴더
```

## 프로그램 정보
- 템플릿: `20260318/TBM_회의록_20260316.xlsx` (복사해서 데이터만 채워넣을 것, 원본 수정 금지)
- 회사명: `(주)아이디에프이앤씨` 고정
- TBM 리더: 참석자 첫 번째 사람
- 직책: 반장

## 자동 생성 항목
- 날짜: 오늘 날짜 자동
- 위험요인 3개: 작업명/작업내용 키워드로 자동 생성
- 대책 3개: 위험요인에 맞는 대책 자동 생성
- 중점위험요인/대책: 자동 생성
- 참석 인원수: 자동 계산

## 사진 처리
- 사진 파일명: `YYYYMMDD.jpg` (예: 20260316.jpg)
- GUI: 오늘 날짜 사진만 자동 선택 (파일명, 수정일, EXIF 기준). 없으면 사진 없이 진행
- 일괄: 날짜별 파일명 매칭

## 사용자 주의사항
- 사용자는 코딩을 모르는 초등학생 수준으로 설명할 것
- 전문 용어 사용 금지
- 단계별로 쉽게 안내할 것
